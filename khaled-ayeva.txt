    1. What is Semantic HTML? 
    Answer: The use of Markup to give meaning to each portion of code.
    
    2. What is HTML used for? 
    Answer: The create and define the structure of websites.
    
    3. What is an attribute and where do we put it? 
    Answer: A portion of syntax that adds specificity to an element.
    
    4. What is the h1 tag used for? How many times should I use it on a page?
    Answer: A header tag usually given to the page title or heading. Ideally this should only be used once.
    
    5. Name two tags that have required attributes
    Answer: Anchor requires Href (<a href="">) / Image requires Source (<img src="">)
    
    6. What do we put in the head of our HTML document? 
    Answer: Title.
    
    7. What is an id? 
    Answer: A unique identifier attached to an element so that it can be identified/affected later.
    
    8. What elements can I add an id to? 
    Answer: Any element.
    
    9. How many times can I use the same id on a page? 
    Answer: Once.
    
    10. What is a class? 
    Answer: An identifier attached to one or more elements so that they can be identified/affected later.
    
    11. What elements can I add a class to? 
    Answer: Any element.
    
    12. How many times can I use the same class on a page? 
    Answer: Unlimited times.
    
    13. How do I get my link to open in a new tab?
    Answer: Add 'target="_blank"' after the link in an Anchor element.
    
    14. What is the alt attribute in the image tag used for? 
    Answer:  To provide alternative info about an image if a user cannot view it.
    
    15. How do I reference an id?
    Answer: '#'
    
    16. What is the difference between a section and a div
    Answer: A section is meant to break up parts of the html the same way a header, body and footer would. 
            Whereas a div is meant to split up the smaller parts of the previously described sections.

    17. What is CSS used for? 
    Answer: It is used to style the structure (html) of a webpage.
    
    18. How to we select an element? Example - every h2 on the page
    Answer: h2 {}
    
    19. What is the difference between a class and an id? - Give me an example of when I might use each one
    Answer: A class is meant to identify a group of elements whereas an id is meant to identify one.
            Ex. A class name: 'Text' to style all text on a page to a shade of grey, but an id: 'Headline' to style the title of an article.

    20. How do we select classes in CSS?
    Answer: A '.' and then the tag (.text).
    
    21. How do we select a p element with a single class of “human””?
    Answer: p.human
    
    22. What is a parent child selector? When would this be useful? 
    Answer: It is a selector meant to allow for specificity in  styling. 
            Ex.: footer img {} (wanting to style only the images in the footer and not the whole page).

    23. How do you select all links within a div with the class of sidebar?
    Answer: div a.sidebar
    
    24. What is a pseudo selector?
    Answer: A class used to indentify an element in a specific state.
    
    25. What do we use the change the spacing between lines?
    Answer: line-spacing: 1px;
    
    26. What do we use to change the spacing between letters?
    Answer: letter-spacing 1px;
    
    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
    Answer: text-transform: capitalize;
    
    28. How do I add a 1px border around my div that is dotted and black?
    Answer: div { border: dotted 10px black}
    
    29. How do I select everything on the page? 
    Answer: * (star)
    
    30. How do I write a comment in CSS?
    Answer: /* Like this */
    
    31. How do I find out what file I am in, when I am using the command line? 
    Answer: pwd (print working directory)
    
    32. Using the command line - how do I see a list of files/folders in my current folder?
    Answer: ls (list)
    
    33. How do I remove a file via the command line? Why do I have to be careful with this? 
    Answer: 'rm'. Should be careful because it's possible to delete system files essential to your system.

    34. Why should I use version control? 
    Answer: It provides a system to track changes and propose new ones to your team.
    
    35. How often should I commit to github?
    Answer: As often as any major change.
    
    36. What is the command we would use to push our repo up to github? 
    Answer: 'git push push -u origin first-name'
    
    37. Walk me through Lambda's git flow. 
    Answer: 
            1. cd — start from main directory
            2. cd Desktop/project
            3. git clone URL
            4. cd (into cloned project name)
            5. git checkout -b 'your name' (Changes branch)
            6. code . (Opens this branch in VSCode— work)
            7. git add . (adds all files)
            8. git commit -m 'message'
            9. git push -u origin 'your name'
